==========
0) Resumen ejecutivo
==========

Este proyecto envia un reporte diario (y/o al arranque, segun timer) a Telegram para confirmar:
- que el servidor esta "vivo"
- si arranco sano (salud) o degradado (alertas)
- que acciones operativas tomar si hay problemas

Objetivo principal: que alguien pueda mirar el chat desde el celular y decidir rapido si hay que actuar.

Incluye:
- Resumen con botones a mensajes de detalle (Temperaturas, Actualizaciones, Servicios, Actualizar)
- Alertas en mensaje separado si hay CRIT (por ejemplo: servicios fallidos)
- Anti-duplicados (no reenvia el mismo dia salvo force)
- Backoff/reintentos para red y API de Telegram
- Hardening basico en systemd (usuario dedicado, RuntimeDirectory, etc.)

==========
1) Diseno
==========

1.1. Componentes

- /opt/boot-report/boot-report.sh
  Script principal (bash). Recolecta senales, decide que mensajes enviar y publica en Telegram.

- /opt/boot-report/lib/
  Librerias bash (render, system, telegram).

- /opt/boot-report/.env
  Secrets + configuracion.

- /etc/systemd/system/boot-report.service
  Servicio oneshot que ejecuta el script.

- /etc/systemd/system/boot-report.timer
  Timer diario (si aplica en tu instalacion).

- /var/lib/boot-report/last_run_date
  Stamp anti-duplicados por dia.

1.2. Flujo (alto nivel)

1) systemd dispara boot-report.service (por timer o manual).
2) boot-report.sh carga .env y valida variables requeridas.
3) Toma lock (flock) para evitar ejecuciones concurrentes.
4) Chequea stamp de "ya se ejecuto hoy".
5) Recolecta metricas (CPU/RAM/Disco/Temp/Updates/Servicios).
6) Envia Telegram:
   - Envia banners (security / critico) si corresponde
   - Envia resumen (con botones)
   - Envia mensajes de detalle como reply al resumen
7) Edita el resumen para agregar botones con links (deep links al chat).
8) Si el envio fue OK: escribe last_run_date.

1.3. Anti-duplicados (lo importante)

Hay 2 mecanismos:
- Lock (flock): evita que se corran dos instancias al mismo tiempo.
- Stamp diario: evita reenviar si el servicio se reinicia o se ejecuta dos veces el mismo dia.

Forzar envio (para pruebas/UI):
- borrar el stamp:
  sudo rm -f /var/lib/boot-report/last_run_date
  sudo systemctl start boot-report.service

:::warn
No confundas "no envia" con "SKIP: ya se ejecuto hoy". Si ves ese log, el script esta funcionando y esta bloqueando el reenvio por diseno.
:::

1.4. Señales y umbrales

:::table
| Senal | Que mide | OK | AVISO | CRITICO | Accion sugerida |
|------|----------|----|------:|--------:|-----------------|
| Carga | load normalizado por cores | < warn | >= warn | >= crit | ver procesos, capacity |
| Memoria | % RAM usada | < warn | >= warn | >= crit | leak/cache, swap |
| Disco | % uso / | < warn | >= warn | >= crit | limpiar, crecer disco |
| Temp | sensores (best-effort) | < warn | >= warn | >= crit | revisar ventilacion/carga |
| Updates | paquetes pendientes | 0 | >0 | n/a | ventana de mantenimiento |
| Servicios | systemd failed units | 0 | n/a | >0 | diagnosticar unit |

Nota: umbrales se controlan por .env (WARN_* / CRIT_*).

2) Requisitos
==========

2.1. Paquetes

```bash
sudo apt update
sudo apt install -y curl ca-certificates
```

Temperaturas (opcional, recomendado):

```bash
sudo apt install -y lm-sensors
sudo sensors-detect --auto || true
```

Red "wait-online" real (recomendado):

- Si usas NetworkManager:
```bash
sudo systemctl enable --now NetworkManager-wait-online.service
```

- Si usas systemd-networkd:
```bash
sudo systemctl enable --now systemd-networkd-wait-online.service
```

==========
3) Instalacion
==========

Hay dos caminos: Git (recomendado) o Manual (copy/paste).

3.1. Camino A - Instalacion via Git (recomendada)

```bash
cd ~
git clone https://github.com/lucasborges2001/Boot.git boot-report
cd boot-report
sudo ./scripts/install.sh
```

Configurar .env:

```bash
sudo nano /opt/boot-report/.env
```

Probar:

```bash
sudo systemctl start boot-report.service
sudo journalctl -u boot-report.service -n 120 --no-pager
```

Actualizar:

```bash
cd ~/boot-report
git pull
sudo ./scripts/install.sh
```

3.2. Camino B - Instalacion manual (copy/paste)

Crear estructura:

```bash
sudo mkdir -p /opt/boot-report/lib
sudo useradd -r -s /usr/sbin/nologin bootreport || true
```

Crear .env:

```bash
sudo nano /opt/boot-report/.env
```

Contenido minimo:

```bash
BOT_TOKEN="PEGAR_TOKEN_TELEGRAM"
CHAT_ID="-1001234567890"
SERVER_LABEL="ServidorCargadores"
```

Permisos:

```bash
sudo chown -R bootreport:bootreport /opt/boot-report
sudo chmod 0750 /opt/boot-report/boot-report.sh
sudo chmod 0640 /opt/boot-report/lib/*.sh
sudo chmod 0640 /opt/boot-report/.env
```

:::warn
Si tu shell es zsh, los comodines pueden fallar con "no matches found". Alternativa robusta:

```bash
sudo find /opt/boot-report/lib -maxdepth 1 -type f -name "*.sh" -exec chmod 0640 {} \;
```
:::

Instalar systemd:

```bash
sudo cp systemd/boot-report.service /etc/systemd/system/boot-report.service
sudo cp systemd/boot-report.timer   /etc/systemd/system/boot-report.timer
sudo systemctl daemon-reload
sudo systemctl enable --now boot-report.timer
```

==========
4) Configuracion (.env)
==========

Obligatorias:
- BOT_TOKEN
- CHAT_ID

Recomendadas:
- SERVER_LABEL
- LOCK_TTL_SEC
- WARN_LOAD_PCT / CRIT_LOAD_PCT
- WARN_RAM_PCT  / CRIT_RAM_PCT
- WARN_DISK_PCT / CRIT_DISK_PCT
- WARN_TEMP_C   / CRIT_TEMP_C

Control de mensajes:
- BOOT_ALERTS_ONLY="false"
  Si es true: si NO hay issues, NO envia.

- BOOT_SMART_CHECK="weekly"
  Controla frecuencia de SMART (none/daily/weekly). Si falla, se reporta.

==========
5) Operacion
==========

5.1. Ver logs

```bash
sudo journalctl -u boot-report.service -n 200 --no-pager
```

Logs tipicos:
- OK: envio exitoso
- FAIL: fallo envio/parametros
- SKIP: ya se ejecuto hoy o ya esta corriendo

5.2. Forzar envio (modo prueba)

```bash
sudo rm -f /var/lib/boot-report/last_run_date
sudo systemctl start boot-report.service
```

5.3. Permisos (regla)

- boot-report.sh debe ser ejecutable:
  0750 bootreport:bootreport

- lib/*.sh y .env NO ejecutables:
  0640 bootreport:bootreport

:::error
Si ves status=203/EXEC, casi siempre es porque boot-report.sh perdio el bit ejecutable (chmod 0640).
:::

==========
6) Troubleshooting (rutas de diagnostico)
==========

6.1. No llega Telegram

1) logs:
```bash
sudo journalctl -u boot-report.service -n 200 --no-pager
```

2) red/dns:
```bash
ip route get 1.1.1.1
getent hosts api.telegram.org
curl -I https://api.telegram.org
```

3) token/chat:
- BOT_TOKEN correcto
- CHAT_ID correcto
- si es grupo: bot agregado y permisos

6.2. Dice SKIP y no envia

- Es anti-duplicados. Forzar envio:
```bash
sudo rm -f /var/lib/boot-report/last_run_date
sudo systemctl start boot-report.service
```

6.3. status=203/EXEC

- Reponer permisos:
```bash
sudo chmod 0750 /opt/boot-report/boot-report.sh
sudo chown bootreport:bootreport /opt/boot-report/boot-report.sh
```

- Opcional (a prueba de permisos): en el unit usar:
  ExecStart=/usr/bin/env bash /opt/boot-report/boot-report.sh

==========
7) Checklist final
==========

:::check
- /opt/boot-report/.env existe (0640) y owner bootreport
- /opt/boot-report/boot-report.sh existe (0750) y owner bootreport
- /opt/boot-report/lib/*.sh existen (0640) y owner bootreport
- boot-report.service funciona manualmente (systemctl start)
- llega mensaje a Telegram
- timer habilitado (si aplica)
- anti-duplicados OK (SKIP cuando corresponde)
:::

==========
8) Desinstalacion / rollback
==========

Detener y deshabilitar:

```bash
sudo systemctl disable --now boot-report.timer boot-report.service
sudo systemctl daemon-reload
```

Remover archivos:

```bash
sudo rm -f /etc/systemd/system/boot-report.service
sudo rm -f /etc/systemd/system/boot-report.timer
sudo rm -rf /opt/boot-report
sudo rm -rf /var/lib/boot-report
```

Usuario dedicado (opcional):

```bash
sudo userdel bootreport 2>/dev/null || true
```
